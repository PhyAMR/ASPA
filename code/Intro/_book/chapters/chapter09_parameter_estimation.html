9  Chapter 9 - Bayesian Parameter Estimation

Conjugate Priors, Posterior Analysis, and Point Estimation

9.1 Learning Objectives

After completing this chapter, you should be able to:

  • Understand the concept of conjugate priors
  • Compute posterior distributions for common models
  • Calculate posterior summary statistics
  • Apply MAP and LMS estimators
  • Understand the reliability of Bayesian estimates

9.2 Overview

Note

This chapter covers material from the lecture slides: Advanced_Statistics_ParameterEstimation.pdf

9.3 Bayes’ Theorem and Bayesian Inference

9.3.1 The Framework

Bayesian inference updates our beliefs about parameters \(\theta\) after observing data \(D\):

\[P(\theta | D, H) = \frac{P(D | \theta, H) P(\theta | H)}{P(D | H)}\]

where: - \(P(\theta | H)\) = Prior (initial belief) - \(P(D | \theta, H)\) = Likelihood (probability of data given parameters) - \(P(\theta | D, H)\) = Posterior (updated belief) - \(P(D | H)\) = Evidence (marginal likelihood, normalizing constant)

9.4 Conjugate Priors

A prior is conjugate to a likelihood if the posterior has the same form as the prior.

9.4.1 Beta-Bernoulli / Beta-Binomial

For estimating a probability \(p\) from \(k\) successes in \(n\) trials:

Prior: \(p \sim \text{Beta}(\alpha, \beta)\)

Likelihood: \(k | p \sim \text{Binomial}(n, p)\)

Posterior: \(p | k \sim \text{Beta}(\alpha + k, \beta + n - k)\)

9.4.2 Posterior Derivation

# Example: Coin flipping
# Prior: Beta(2, 2) - slightly biased toward fairness
# Data: 7 heads out of 10 flips

alpha_prior <- 2
beta_prior <- 2
k <- 7  # heads
n <- 10  # total flips

# Posterior
alpha_post <- alpha_prior + k
beta_post <- beta_prior + n - k

cat("Posterior: Beta(", alpha_post, ",", beta_post, ")\n")
Posterior: Beta( 9 , 5 )
# Plot prior and posterior
p <- seq(0, 1, length.out = 200)
prior <- dbeta(p, alpha_prior, beta_prior)
posterior <- dbeta(p, alpha_post, beta_post)

plot(p, prior, type = "l", col = "blue", lty = 2,
     ylab = "Density", xlab = "p",
     main = "Beta-Binomial: Prior and Posterior")
lines(p, posterior, col = "red", lwd = 2)
abline(v = k/n, col = "green", lty = 3)  # MLE (sample proportion)
legend("topright", legend = c("Prior", "Posterior", "MLE"),
       col = c("blue", "red", "green"), lty = c(2, 1, 3), lwd = c(1, 2, 1))

9.4.3 Effect of Sample Size

# Show how posterior concentrates as n increases
p <- seq(0, 1, length.out = 1000)
true_p <- 0.7
n_values <- c(10, 50, 200)

# Uniform prior
alpha <- 1
beta <- 1

colors <- c("red", "blue", "darkgreen")
plot(0, 0, xlim = c(0, 1), ylim = c(0, 15), type = "n",
     xlab = "p", ylab = "Density",
     main = "Posterior Concentration with Increasing Data")

for (i in seq_along(n_values)) {
  n <- n_values[i]
  k <- round(true_p * n)
  posterior <- dbeta(p, alpha + k, beta + n - k)
  lines(p, posterior, col = colors[i], lwd = 2)
}
abline(v = true_p, col = "black", lty = 2)
legend("topright", legend = c(paste0("n=", n_values), "True p"),
       col = c(colors, "black"), lty = c(1, 1, 1, 2), lwd = c(2, 2, 2, 1))

9.5 Posterior Summary Statistics

9.5.1 Posterior Mean (LMS - Least Mean Square)

For Beta\((\alpha, \beta)\):

\[\mathbb{E}[p | k, n] = \frac{\alpha + k}{\alpha + \beta + n}\]

As \(n \to \infty\): \(\mathbb{E}[p | \text{data}] \to \frac{k}{n}\) (empirical frequency)

9.5.2 Posterior Mode (MAP - Maximum A Posteriori)

\[p_{\text{MAP}} = \frac{\alpha + k - 1}{\alpha + \beta + n - 2}\]

(Valid when \(\alpha + k > 1\) and \(\beta + n - k > 1\))

9.5.3 Posterior Variance

\[\text{Var}(p | k, n) = \frac{(\alpha + k)(\beta + n - k)}{(\alpha + \beta + n)^2(\alpha + \beta + n + 1)}\]

As \(n \to \infty\): variance \(\to 0\) (posterior concentrates)

9.5.4 In R

# Calculate posterior statistics
alpha <- 7   # from Beta(7, 5) posterior
beta <- 5

# Posterior mean
post_mean <- alpha / (alpha + beta)
cat("Posterior mean:", post_mean, "\n")
Posterior mean: 0.5833333 
# Posterior mode
post_mode <- (alpha - 1) / (alpha + beta - 2)
cat("Posterior mode (MAP):", post_mode, "\n")
Posterior mode (MAP): 0.6 
# Posterior variance
post_var <- alpha * beta / ((alpha + beta)^2 * (alpha + beta + 1))
cat("Posterior variance:", post_var, "\n")
Posterior variance: 0.01869658 
cat("Posterior std:", sqrt(post_var), "\n")
Posterior std: 0.1367354

9.6 Point Estimation

9.6.1 General Framework

We often want a single estimate \(\hat{\theta}\) to quote from the posterior.

The optimal estimate depends on the loss function \(L(a, \theta)\):

\[\hat{\theta}(X) = \arg\min_a \mathbb{E}[L(a, \theta) | X]\]

9.6.2 MAP Estimator (0-1 Loss)

For 0-1 loss: \(L(a, \theta) = \begin{cases} 0 & a = \theta \\ 1 & a \neq \theta \end{cases}\)

\[\hat{\theta}_{\text{MAP}} = \arg\max_\theta p(\theta | X)\]

Pros: - Simple, computationally convenient - Natural interpretation (most probable value)

Cons: - Uses only local information (mode may not be representative) - Not invariant under reparameterization

9.6.3 LMS Estimator (Squared Loss)

For squared loss: \(L(a, \theta) = (a - \theta)^2\)

\[\hat{\theta}_{\text{LMS}} = \mathbb{E}[\theta | X]\]

Pros: - Uses full posterior distribution - Invariant under reparameterization

Cons: - Sensitive to tails - May not be the “most likely” value

9.6.4 Comparison

# Example where MAP and mean differ
# Bimodal posterior
p <- seq(0, 1, length.out = 500)
posterior <- 0.6 * dbeta(p, 2, 8) + 0.4 * dbeta(p, 8, 2)

# Normalize
posterior <- posterior / sum(posterior) * length(p)

# Find MAP
map_idx <- which.max(posterior)
map_estimate <- p[map_idx]

# Approximate mean
mean_estimate <- sum(p * posterior) / sum(posterior)

plot(p, posterior, type = "l", ylab = "Density",
     main = "Bimodal Posterior: MAP vs Mean")
abline(v = map_estimate, col = "red", lty = 2, lwd = 2)
abline(v = mean_estimate, col = "blue", lty = 3, lwd = 2)
legend("topright", legend = c("Posterior", "MAP", "Mean"),
       col = c("black", "red", "blue"), lty = c(1, 2, 3), lwd = c(1, 2, 2))

9.7 Gaussian Approximation

For large samples, the posterior can be approximated by a Gaussian centered at the MAP.

9.7.1 Laplace Approximation

Taylor expansion of log-posterior around MAP:

\[\mathcal{L}(\theta) \approx \mathcal{L}(\theta_{\text{MAP}}) - \frac{1}{2}(\theta - \theta_{\text{MAP}})^T H (\theta - \theta_{\text{MAP}})\]

where \(H\) is the negative Hessian matrix of log-posterior.

Posterior \(\approx \mathcal{N}(\theta_{\text{MAP}}, C)\) where \(C = H^{-1}\)

9.7.2 Final Estimate

\[\hat{\theta} = \theta_{\text{MAP}} \pm \sqrt{\text{diag}(C)}\]

9.7.3 Bernoulli Example

# Gaussian approximation for Beta posterior
alpha <- 12
beta <- 8
n <- alpha + beta - 2  # effective sample size
k <- alpha - 1

# MAP
map <- (alpha - 1) / (alpha + beta - 2)

# Approximate variance (inverse Fisher information)
# For Bernoulli: var ≈ p(1-p)/n
approx_var <- map * (1 - map) / n

p <- seq(0, 1, length.out = 200)
exact <- dbeta(p, alpha, beta)
gaussian_approx <- dnorm(p, mean = map, sd = sqrt(approx_var))

plot(p, exact, type = "l", ylab = "Density",
     main = "Beta Posterior vs Gaussian Approximation")
lines(p, gaussian_approx, col = "red", lty = 2)
legend("topright", legend = c("Exact (Beta)", "Gaussian Approx"),
       col = c("black", "red"), lty = c(1, 2))

9.8 Loss and Risk

9.8.1 Posterior Risk

The expected loss given the data:

\[\rho(a | x) = \int L(a, \theta) p(\theta | x) d\theta\]

9.8.2 Bayes Risk

The overall quality of an estimator:

\[\mathcal{R} = \int \rho(\hat{\theta}(x) | x) p(x) dx\]

9.8.3 Key Result

Minimizing the posterior risk for each \(x\) automatically minimizes the global Bayes risk.

Local optimum = Global optimum

9.9 Take-Home Messages

  1. Bayesian estimation minimizes expected loss given the data
  2. The local decision rule is already globally optimal
  3. Conjugate priors make computation tractable
  4. Posterior concentrates as data grows (consistency)
  5. MAP is simple but not always representative; Mean uses full posterior

9.10 Summary

Estimator Loss Function Formula Properties
MAP 0-1 \(\arg\max p(\theta | x)\) Simple, not invariant
LMS Squared \(\mathbb{E}[\theta | x]\) Uses full posterior, invariant

Key takeaways from this chapter:

  1. Conjugate priors: Prior and posterior have same form (Beta-Binomial, Gamma-Poisson, etc.)
  2. MAP: Maximum a posteriori - simplest point estimate
  3. LMS: Posterior mean - uses full distribution
  4. Gaussian approximation: For large samples, posterior ≈ Gaussian centered at MAP
  5. Loss functions: Define what “good” estimate means; squared loss gives mean, 0-1 gives mode

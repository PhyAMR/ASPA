10  Chapter 10 - Bayesian Model Comparison

Bayes Factors, Occam’s Razor, and Model Selection

10.1 Learning Objectives

After completing this chapter, you should be able to:

  • Calculate and interpret Bayes factors
  • Understand Occams razor in the context of Bayesian inference
  • Compare models using evidence
  • Distinguish between Bayesian and Frequentist hypothesis testing
  • Apply model comparison to practical problems

10.2 Overview

This chapter covers material from the lecture slides: Advanced_Statistics_model_comparison.pdf

10.3 Bayes Factors

10.3.1 Comparing Hypotheses

The Bayes factor compares evidence for two hypotheses:

\[R = \frac{P(H_2 | D)}{P(H_1 | D)} = \frac{P(D | H_2)}{P(D | H_1)} \times \frac{P(H_2)}{P(H_1)}\]

The Bayes factor (BF) is the ratio of marginal likelihoods:

\[\text{BF}_{21} = \frac{P(D | H_2)}{P(D | H_1)}\]

10.3.2 Interpretation

Bayes Factor Evidence for \(H_2\)
1-3 Not worth more than a bare mention
3-20 Positive
20-150 Strong
> 150 Very strong

10.3.3 Evidence (Marginal Likelihood)

\[P(D | H) = \int P(D | \theta, H) P(\theta | H) d\theta\]

The evidence: 1. Normalizes the posterior 2. Enables model comparison 3. Marginalizes over parameters

10.4 Occam’s Razor

10.4.1 The Principle

“Pluralitas non est ponenda sine necessitate” (Entities should not be multiplied without necessity).

In Bayesian terms: Models that are too complex are automatically penalized.

10.4.2 Laplace Approximation

Assuming the likelihood is peaked at \(\hat{\theta}\) with width \(\Delta\theta_{\text{eff}}\):

\[P(D | H) \approx \underbrace{P(D | \hat{\theta}, H)}_{\text{best fit}} \times \underbrace{\pi(\hat{\theta}) \Delta\theta_{\text{eff}}}_{\text{Occam factor}}\]

Key idea: Complex models spread prior mass over a large parameter space, so only a small region fits the data.

10.4.3 Toy Example: Sequence Prediction

Consider the sequence \(D = (-1, 3, 7, 11)\). What comes next?

Two hypotheses: - \(H_1\): Linear (arithmetic progression), 1 parameter - \(H_2\): Cubic, 3 parameters

Both fit the data perfectly! But which is better?

# Generate the two predictions
n <- 1:6

# H1: Linear (y_{n+1} = y_n + 4)
y_H1 <- -1 + 4*(n - 1)

# H2: Cubic (complex recurrence)
y_H2 <- c(-1, 3, 7, 11, -19.9, 1043.8)

# Plot
plot(1:4, y_H2[1:4], pch = 16, xlim = c(1, 6), ylim = c(-20, 50),
     xlab = "n", ylab = "y",
     main = "Model Comparison: Linear vs Cubic")
points(5:6, y_H2[5:6], pch = 17, col = "red")
lines(n, y_H1, col = "blue", lty = 2)
legend("topleft", legend = c("Data", "H2 predictions", "H1 (linear)"),
       pch = c(16, 17, NA), col = c("black", "red", "blue"),
       lty = c(NA, NA, 2))

The cubic model (\(H_2\)) makes wild predictions, while the linear model (\(H_1\)) gives sensible extrapolation.

10.5 Model Comparison

10.5.1 Comparing Models with Equal Priors

If \(P(H_1) = P(H_2)\):

\[R = \text{BF} = \frac{P(D | H_2)}{P(D | H_1)}\]

10.5.2 Occam Factor

When both models fit equally well at their best parameters:

\[\text{BF} \approx \frac{\pi_2(\hat{\theta}_2) \Delta\theta_2}{\pi_1(\hat{\theta}_1) \Delta\theta_1} = \frac{\text{Occam}_2}{\text{Occam}_1}\]

The Occam factor = fraction of parameter space that fits the data.

10.5.3 Estimating Occam Factors

For uniform priors over parameter ranges of size \(L\):

\[\text{Occam} \approx \frac{\Delta\theta}{L}\]

For a polynomial model with parameters \(\theta_k\): - \(\Delta\theta_k \sim \frac{\sigma}{n^k}\) - Higher-order parameters must be finely tuned

10.5.4 Example Calculation

Simple model (\(H_1\), 1 parameter): \[\Delta d \sim \frac{\sigma}{N} \Rightarrow \text{Occam}_1 \sim \frac{\sigma}{NL}\]

Cubic model (\(H_2\), 3 parameters): \[\Delta\alpha \sim \frac{\sigma}{N^3}, \Delta\beta \sim \frac{\sigma}{N^2}, \Delta\gamma \sim \frac{\sigma}{N}\] \[\text{Occam}_2 \sim \frac{\sigma^3}{L^3 N^6}\]

Ratio: \[R \sim \frac{\text{Occam}_2}{\text{Occam}_1} \sim \frac{\sigma^2}{L^2 N^5}\]

For typical values, this \(\ll 1\), strongly favoring the simpler model.

10.6 Physical Interpretation

10.6.1 Simple Model (\(H_1\)):

  • Many parameter values give similar predictions
  • Stable and predictive
  • Robust to perturbations

10.6.2 Complex Model (\(H_2\)):

  • Requires fine-tuning of parameters
  • Fits data as a coincidence
  • Most of parameter space doesnt explain the data

Conclusion: Complex models are penalized because most of their parameter space doesnt explain the data.

10.7 Frequentist Hypothesis Testing

10.7.1 Likelihood Ratio Test

Model: \((\Omega, \mathcal{F}, \mathbb{P}_\theta)\), \(\theta \in \Theta\)

Hypotheses: - \(H_0: \theta \in \Theta_0\) - \(H_1: \theta \in \Theta_1\)

Test Statistic: \[\Lambda(X) = \frac{\text{best fit under } H_0}{\text{best fit overall}} = \frac{\sup_{\theta \in \Theta_0} \mathbb{P}_\theta(X)}{\sup_{\theta \in \Theta} \mathbb{P}_\theta(X)}\]

Interpretation: - \(\Lambda \approx 1\): Restricted model explains as well as full one - \(\Lambda \ll 1\): Full model explains much better (evidence against \(H_0\))

10.7.2 p-values

Log-likelihood difference: \[T(X) = -2\ln\Lambda(X) = 2(\ln L_{\text{full}} - \ln L_{H_0})\]

Under \(H_0\), \(T(X) \sim \chi^2_k\) where \(k\) = difference in dimensions.

p-value: \[p(X_{\text{obs}}) = \sup_{\theta \in \Theta_0} \mathbb{P}_\theta(T(X) \geq T(X_{\text{obs}}))\]

Decision: Reject \(H_0\) if \(p < \alpha\).

10.7.3 Key Differences with Bayesian Approach

Aspect Frequentist Bayesian
Probability of hypotheses Not defined Well-defined
Evidence p-values, likelihood ratio Bayes factor
Complexity Manual (AIC, BIC) Automatic (Occam factor)
Decision Binary (reject/fail to reject) Probabilistic

10.8 Bayesian vs Frequentist

10.8.1 Bayesian:

  • Assign probabilities to hypotheses
  • Use Bayes factor
  • Automatically penalizes complexity (Occam)
  • Requires specification of prior

10.8.2 Frequentist:

  • No probability for hypotheses
  • Use p-values and tests
  • Focus on error control (Type I, Type II)
  • Complexity penalties added manually

10.9 Generative Models

10.9.1 Definition

A generative model predicts observable data from parameters.

Simplest example: Linear model \[f(x; a, b) = a + bx\]

Measurements include noise: \[y = f(x; \theta) + \varepsilon\]

where: - \(\theta_f\): model parameters - \(\varepsilon\): measurement noise (often \(\varepsilon \sim \mathcal{N}(0, \sigma^2)\))

10.9.2 Gaussian Measurement Model

\[P(y | \theta, H) = \frac{1}{\sigma\sqrt{2\pi}} \exp\left[-\frac{(y - f(x; a, b))^2}{2\sigma^2}\right]\]

Interpretation: - Measurements fluctuate around model prediction - Width controlled by \(\sigma\)

10.9.3 Likelihood for Dataset

Assuming independent measurements:

\[P(D | \theta, H) = \prod_j P(y_j | x_j, \theta, H)\]

For Gaussian noise:

\[P(D | \theta, H) \propto \exp\left[-\frac{1}{2\sigma^2} \sum_j (y_j - f(x_j; \theta))^2\right]\]

Connection to least squares: \[\chi^2 = \sum_j (y_j - f(x_j; \theta))^2\]

10.9.4 Evidence and Computational Aspects

The evidence (marginal likelihood): \[P(D | H) = \int P(D | \theta, H) P(\theta | H) d\theta\]

Roles: - Normalization of posterior - Model comparison - Marginalization

Computational Challenge: - Analytical solutions only in simple cases - High-dimensional integrals - Complex likelihoods

Standard approach: Markov Chain Monte Carlo (MCMC)

10.10 Summary of Key Results

  1. Generative model + noise defines likelihood
  2. Bayesian inference updates prior using data
  3. Posterior contains full parameter information
  4. Evidence enables normalization and model comparison
  5. Numerical methods often required

10.11 Summary

Key takeaways from this chapter:

  1. Bayes Factor: Ratio of marginal likelihoods, quantifies evidence
  2. Occams Razor: Complex models are automatically penalized
  3. Occam Factor: Fraction of prior mass consistent with data
  4. Bayesian vs Frequentist: Different frameworks with different interpretations
  5. Model Selection: Balance fit quality with model complexity

10.12 Exercises

Exercise 1

Two models both fit the data equally well at their best parameters. Model A has 1 parameter with Occam factor \(10^{-3}\). Model B has 3 parameters with Occam factor \(10^{-9}\) for each parameter. Calculate the Bayes factor and determine which model is favored.

Exercise 2

Explain the statement “Fit quality is not enough. Model complexity matters.” Give a concrete example where a more complex model fits the training data better but generalizes worse.

Exercise 3

Compare the Bayesian approach to model selection (Bayes factors) with the frequentist approach (p-values, likelihood ratio test). What are the advantages and disadvantages of each?

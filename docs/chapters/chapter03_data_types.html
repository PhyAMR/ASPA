3  Chapter 3 - Data Types and Precision in R

Understanding R Data Types and Floating Point Arithmetic

3.1 Learning Objectives

After completing this chapter, you should be able to:

  • Understand the different atomic data types in R
  • Recognize issues with floating point precision
  • Use type coercion appropriately
  • Handle special values (NA, NaN, Inf) correctly

3.2 Overview

This chapter covers material from the lecture slides: Advanced_Statistics_R_third.pdf

3.3 Atomic Data Types in R

R has six basic atomic vector types:

Type Mode Description
logical logical TRUE/FALSE values
integer numeric Whole numbers (L suffix)
double numeric Real numbers
character character Text strings
complex complex Complex numbers
raw raw Raw bytes

3.3.1 Logical Type

# Logical values
l1 <- TRUE
l2 <- FALSE

# Logical operations
!l1           # Negation
[1] FALSE
l1 & l2       # AND
[1] FALSE
l1 | l2       # OR
[1] TRUE
xor(l1, l2)   # Exclusive OR
[1] TRUE
# Comparison operators return logical
x <- 5
x < 10       # TRUE
[1] TRUE
x >= 5       # TRUE
[1] TRUE
x == 5        # TRUE
[1] TRUE
x != 5        # FALSE
[1] FALSE
# Any and All
any(c(FALSE, TRUE, FALSE))   # TRUE
[1] TRUE
all(c(TRUE, TRUE, TRUE))       # TRUE
[1] TRUE

3.3.2 Numeric Types: Integer and Double

# Integer (explicit with L suffix)
int_var <- 42L
class(int_var)
[1] "integer"
typeof(int_var)
[1] "integer"
# Double (default for numbers with decimal point)
dbl_var <- 42.0
class(dbl_var)
[1] "numeric"
typeof(dbl_var)
[1] "double"
# Even integers stored as double by default
num <- 42
class(num)    # "numeric" (which means double)
[1] "numeric"
# Converting between types
as.integer(3.14)    # 3 (truncates)
[1] 3
as.double(42L)      # 42.0
[1] 42

3.3.3 Character Type

# Character strings
str1 <- "Hello"
str2 <- 'World'  # Single quotes also work

# String concatenation
paste(str1, str2)           # "Hello World"
[1] "Hello World"
paste0(str1, str2)          # "HelloWorld"
[1] "HelloWorld"
# String manipulation
toupper("hello")            # "HELLO"
[1] "HELLO"
tolower("WORLD")            # "world"
[1] "world"
substr("Hello World", 1, 5) # "Hello"
[1] "Hello"
nchar("Hello")              # 5
[1] 5
# Converting to character
as.character(42)            # "42"
[1] "42"
as.character(3.14159)       # "3.14159"
[1] "3.14159"

3.3.4 Complex Numbers

# Complex numbers
z <- 3 + 4i
class(z)
[1] "complex"
# Operations on complex numbers
Mod(z)          # Modulus (5)
[1] 5
Arg(z)          # Argument (angle)
[1] 0.9272952
Re(z)           # Real part
[1] 3
Im(z)           # Imaginary part
[1] 4
Conj(z)         # Complex conjugate
[1] 3-4i
# Creating complex numbers
complex(real = 3, imaginary = 4)
[1] 3+4i

3.4 Type Coercion

R follows a coercion hierarchy when combining different types:

logical → integer → double → character

# Implicit coercion
mixed1 <- c(1, 2, TRUE, FALSE)      # All become numeric: 1, 2, 1, 0
mixed1
[1] 1 2 1 0
class(mixed1)
[1] "numeric"
mixed2 <- c(1, 2, "three")          # All become character
mixed2
[1] "1"     "2"     "three"
class(mixed2)
[1] "character"
mixed3 <- c(TRUE, 2.5, "text")     # All become character
mixed3
[1] "TRUE" "2.5"  "text"

3.4.1 Explicit Coercion

# Explicit type conversion
as.logical(c(0, 1, 2, -1))    # FALSE, TRUE, TRUE, TRUE
[1] FALSE  TRUE  TRUE  TRUE
as.integer(c(1.9, 2.1, 2.9))  # 1, 2, 2 (truncates!)
[1] 1 2 2
as.numeric(c(TRUE, FALSE))      # 1, 0
[1] 1 0
# Coercion may produce NA
as.numeric(c("1", "2", "three"))  # 1, 2, NA
Warning: NAs introduced by coercion
[1]  1  2 NA
Warning

Be careful with as.integer() - it truncates (rounds toward zero), not rounds! Use round() first if you need proper rounding.

3.5 Floating Point Precision Issues

3.5.1 The Problem

Computers use binary floating point representation, which cannot exactly represent many decimal numbers:

# Classic example
0.1 + 0.2 == 0.3
[1] FALSE
# What is the actual value?
print(0.1 + 0.2, digits = 20)
[1] 0.30000000000000004441
print(0.3, digits = 20)
[1] 0.2999999999999999889
# Machine epsilon (smallest number that changes 1 when added)
.Machine$double.eps
[1] 2.220446e-16
# Comparing floating point numbers
x <- 0.1 + 0.2
y <- 0.3

# Don't do this
x == y    # FALSE!
[1] FALSE
# Do this instead
abs(x - y) < 1e-10    # TRUE
[1] TRUE
# Or use all.equal
all.equal(x, y)        # TRUE
[1] TRUE

3.5.2 Understanding Machine Precision

# Binary representation limitations
for (i in 1:10) {
  print(paste("1/", i, "=", 1/i, "exact:", (i * (1/i)) == 1))
}
[1] "1/ 1 = 1 exact: TRUE"
[1] "1/ 2 = 0.5 exact: TRUE"
[1] "1/ 3 = 0.333333333333333 exact: TRUE"
[1] "1/ 4 = 0.25 exact: TRUE"
[1] "1/ 5 = 0.2 exact: TRUE"
[1] "1/ 6 = 0.166666666666667 exact: TRUE"
[1] "1/ 7 = 0.142857142857143 exact: TRUE"
[1] "1/ 8 = 0.125 exact: TRUE"
[1] "1/ 9 = 0.111111111111111 exact: TRUE"
[1] "1/ 10 = 0.1 exact: TRUE"
# Powers of 2 are exact
1/2 == 0.5      # TRUE
[1] TRUE
1/4 == 0.25     # TRUE
[1] TRUE
1/8 == 0.125    # TRUE
[1] TRUE
# Others are approximate
1/10 == 0.1     # FALSE (in practice, may be TRUE due to rounding)
[1] TRUE

3.5.3 Best Practices for Floating Point

# 1. Use all.equal for comparisons
x <- 1.0000001
y <- 1.0
all.equal(x, y, tolerance = 1e-6)
[1] TRUE
# 2. Be careful with cumulative operations
sum_rep <- function(n, x) {
  total <- 0
  for (i in 1:n) {
    total <- total + x
  }
  return(total)
}

# Compare
n <- 100000
x <- 0.1
sum_rep(n, x) == n * x    # May be FALSE
[1] FALSE
all.equal(sum_rep(n, x), n * x)
[1] TRUE
# 3. Use integer arithmetic when possible
sum_int <- function(n) {
  total <- 0L
  for (i in 1L:n) {
    total <- total + i
  }
  return(total)
}

sum_int(100)
[1] 5050

3.6 Special Values

3.6.1 NA (Not Available)

# NA represents missing values
x <- c(1, 2, NA, 4, 5)
x
[1]  1  2 NA  4  5
# Propagation of NA
mean(x)              # NA (by default)
[1] NA
mean(x, na.rm = TRUE)  # 3 (ignore NA)
[1] 3
# Checking for NA
is.na(x)             # FALSE FALSE TRUE FALSE FALSE
[1] FALSE FALSE  TRUE FALSE FALSE
any(is.na(x))        # TRUE
[1] TRUE
# Removing NA
x[!is.na(x)]         # 1 2 4 5
[1] 1 2 4 5
na.omit(x)           # removes NA values
[1] 1 2 4 5
attr(,"na.action")
[1] 3
attr(,"class")
[1] "omit"

3.6.2 NaN (Not a Number)

# NaN results from undefined operations
0/0                  # NaN
[1] NaN
Inf - Inf            # NaN
[1] NaN
sqrt(-1)             # NaN (for real sqrt)
Warning in sqrt(-1): NaNs produced
[1] NaN
# Checking for NaN
is.nan(NaN)          # TRUE
[1] TRUE
is.nan(NA)           # FALSE
[1] FALSE
is.na(NaN)           # TRUE (NaN is also NA)
[1] TRUE
# NaN propagation
NaN + 5              # NaN
[1] NaN
mean(c(1, 2, NaN))   # NaN
[1] NaN

3.6.3 Inf and -Inf

# Infinity results from overflow
1/0                  # Inf
[1] Inf
-1/0                 # -Inf
[1] -Inf
1e308 * 10           # Inf (overflow)
[1] Inf
# Working with infinity
Inf + 1              # Inf
[1] Inf
Inf - Inf            # NaN
[1] NaN
Inf / Inf            # NaN
[1] NaN
# Checking for infinity
is.infinite(Inf)     # TRUE
[1] TRUE
is.finite(Inf)       # FALSE
[1] FALSE
is.finite(1)         # TRUE
[1] TRUE

3.6.4 NULL

# NULL represents "nothing" or "undefined"
y <- NULL

# NULL vs NA
length(NULL)         # 0
[1] 0
length(NA)           # 1
[1] 1
# Checking for NULL
is.null(NULL)        # TRUE
[1] TRUE
is.null(NA)          # FALSE
[1] FALSE

3.7 Type Testing Functions

x <- 42L
y <- 42.0
z <- "42"

# Testing types
is.integer(x)        # TRUE
[1] TRUE
is.numeric(x)        # TRUE (integer is numeric)
[1] TRUE
is.double(y)         # TRUE
[1] TRUE
is.character(z)      # TRUE
[1] TRUE
# Testing special values
is.na(c(1, NA, 3))           # FALSE TRUE FALSE
[1] FALSE  TRUE FALSE
is.nan(c(1, NaN, 3))         # FALSE TRUE FALSE
[1] FALSE  TRUE FALSE
is.finite(c(1, Inf, -Inf))   # TRUE FALSE FALSE
[1]  TRUE FALSE FALSE
is.infinite(c(1, Inf))       # FALSE TRUE
[1] FALSE  TRUE

3.8 Summary

Key takeaways from this chapter:

  1. Data Types: R has logical, integer, double, character, complex, and raw atomic types
  2. Coercion: Types combine following a hierarchy; explicit coercion with as.*() functions
  3. Floating Point: Binary representation cannot exactly store many decimal values
  4. Comparisons: Use all.equal() instead of == for floating point comparisons
  5. Special Values: NA (missing), NaN (undefined), Inf (overflow), NULL (empty)

Exercise 2
